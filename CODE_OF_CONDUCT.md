# Code of Conduct

## Our Pledge

We want PurpleBear to be a respectful and practical open source project. Everyone participating in this project is expected to communicate clearly, assume good intent, and help keep discussions focused on improving the software.

## Expected Behavior

- Be respectful of different backgrounds, languages, skill levels, and viewpoints.
- Keep issue and pull request discussions technical, specific, and actionable.
- Report bugs with enough detail for maintainers to reproduce them.
- Give feedback on code and design decisions without personal attacks.
- Respect user privacy and avoid sharing private server, subscription, token, or account details.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Posting private credentials, subscription URLs, device identifiers, or logs that expose personal data.
- Spamming issues, pull requests, discussions, or release pages.
- Encouraging illegal, abusive, or harmful use of the project.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants when behavior harms the project or its users. Serious security or privacy concerns should be reported through the security policy instead of public issues.

## Scope

This code of conduct applies to this repository, issues, pull requests, releases, discussions, and any official project communication channel.
